# Creating the VM and installing OPNSense

With the subnets established, it's time to add a firewall to the network to manage them. I created a new VM in virt-manager, assigned it 4GB of memory, 20 GB of storage and 2 cores of CPU. I left the default network interface to maintain internet access and added interfaces for the rest of the subnets. When asked to choose an OS, I did not have FreeBSD options which would have been the most appropriate, but I chose the latest generic Linux option which worked well.

# Configuring OPNSense

After OPNSense booted, and I was presented with a login screen, I logged in with `installer:opnsense` and used all default settings on the installer. I then changed the root password and completed the installation and rebooted.

![](/assets/img/ad_lab/part_two/opnsense_menu.png)

When OPNSense came back up, I logged in as root and used option 1 to Assign interfaces. I made vtnet0 OPNSense's WAN and the LAN was vtnet1. I entered the rest in numerical order as the order of these isn't super important.

When I reached the main menu again, I selected option 2 set interface IP addresses. All Interfaces had DHCP enabled except for the internal one which has the active directory domain on it because the DC will handle the IP assignment.

![](/assets/img/ad_lab/part_two/opnsense_configured.png)

I was then able to access the web interface using https://192.168.100.1 and set names for the interfaces.

# Creating the management VPN

I created a new WireGuard instance and gave it the 10.0.99.1/24 IP range. I used a key pair generated by clicking the gear button.

When the instance was created, an accompanying virtual interface was created that needs to be assigned and enabled. The WAN interface also needs to have the boxes unchecked for blocking RFC1918 private addresses and bogon networks.

# Adding firewall rules to allow the VPN traffic

By default, all traffic between networks is blocked by OPNSense, so I need to add a few rules to enable a VPN. The first is to allow the handshake, so I create a rule to pass traffic from the WAN to the OPNSense WAN IP address over UDP port 51820.

I also created a rule to allow all traffic from the management network to the management VPN network and traffic from the VPN to the management network.

![](/assets/img/ad_lab/part_two/mgmt_vpn_rules.png)

# Adding a peer to the VPN

The WireGuard peer generator makes configuring new peers simple. After plugging values into fields, a config file is generated for me. 

![](/assets/img/ad_lab/part_two/mgmt_vpn_peer.png)

After I installed WireGuard on the host machine, I created a file at /etc/wireguard/wg0.conf and populated its contents with the config file. I then ran `sudo wg-quick up wg0.conf` to connect and verified it by running `sudo wg show` which had a `latest handshake` in the output showing that it was successful. As further proof that it is working, I am also able to ping 192.168.100.1 and I'm even able to load the web interface at the same IP.
